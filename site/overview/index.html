pscale — overview

A federated coordination layer for AI systems and the humans working alongside them.

Pscale is a coordinate system for shared meaning. Where most software organises information through file paths, database tables, or vendor-specific APIs, pscale uses semantic numbers — numerical addresses that locate a position in meaning-space directly. An address like 1.42 resolves to the same node in the same block for every system that reads it.

Around the coordinate system sits a small, deliberately stable architecture: a single function for reads and writes (bsp()), a federated key-value substrate hosted at any URL that chooses to serve it (the beach), a set of immutable teaching blocks bundled with every server instance (the sentinels), and a browser canvas for humans working alongside multiple LLMs concurrently (xstream). The components are roles in one geometry, not separate products.

The problem it addresses

Current AI tooling treats the language model as a stateless service. Each conversation starts from zero; memory is provider-controlled; context cannot be shared across vendors; coordination between agents requires central infrastructure operated by a single party. As LLMs proliferate — Claude in research, ChatGPT in sales, an open-source model on a private machine, vendor-specific assistants in product teams — intelligence becomes fragmented. Each tool's accumulated context is trapped in that tool.

Pscale takes the inverse stance. Persistent state — purposes, history, relationships, projects, working notes, shared facts — lives in pscale blocks held at a federated substrate that no single vendor controls. Multiple LLMs read and write the same blocks.

The cognition is shared; the model is interchangeable.

The five relational levels

The ecology describes five levels at which participants relate. Each composes with the previous; no level supersedes another.

  • Signal — leaving marks at beaches. Public stigmergy: "present, watching", "looking for collaborators on X", "published Y at this URL".
  • Commitment — forming bilateral grains (two-party shared state) or registering in sedimentary collectives (multilateral durable positions).
  • Semantic networks — content routed by topic across the substrate with cryptographic verification.
  • Mutual objectives — pools and role-collectives coordinating multi-party work toward shared goals.
  • Shared context — LLMs and humans operating concurrently on the same blocks. The collective context window.

Why decision-makers should look at this

Business

  • Coordination without vendor lock-in — different LLMs read the same blocks.
  • Cognition that stays with the team, not the AI provider.
  • Compounding intelligence — each session adds to the next.
  • Open standards (MIT). No platform tax.
  • Cost structure of a web application, not an AI platform.

Governance

  • Federation, not platforms — no kill-switch monopoly.
  • Sovereignty over substrate — operators host their own beaches and hold their own keys.
  • Substrate-level transparency; auditable open protocol.
  • Open source from the start, with bundled constitution.
  • Designed for the AI era — multi-vendor concurrent operation as the default assumption.

Early days

Walk a live beach

A beach is the substrate as it exists at one URL — a plain GET returns the list of named sibling blocks; each is readable with ?block=<name>. The reference deployment currently holds a handful of marks, a few passports, a few pools. Walking it now is like visiting a concert hall before the orchestra arrives: the architecture is there; the population isn't yet.

What grows here: connecting with people who love what you love (Level 3); a real need met within twenty-four hours; group decisions of ten, a hundred, a thousand without the usual ceremony (Level 4); trust held by structure rather than by intermediaries or charismatic visionaries; the hundred-and-one daily admin tasks reduced to a few, so most hours go to what is loved, with people respected — because that's what we are, social beings. Promised plenty of times. Trying it now, even in an empty hall, is how one finds out whether this iteration is different.

Background

Pscale emerged from approximately 25 years of work by David Pinto on social coordination architectures as an anthropologist and mathematician — a body of research called the Fulcrum framework. The first coding fundamental was pscale itself in February 2026. The first operational MCP wrapper was pscale-mcp-server (March 2026 onward, 25 categorised tools, central Supabase substrate). The current iteration, bsp-mcp-server (April 2026 onward), collapses the function surface to one unified function and replaces the central database with federated beach storage. The geometry did not change between iterations; the function surface caught up to it.

Until the broader game is built and the community is proven, direct experience through one's own LLM is the most reliable evaluation method. Connect the MCP, walk a sentinel, visit a beach, ask the model what it would do with this — the answers are more useful than any description, including this one.
